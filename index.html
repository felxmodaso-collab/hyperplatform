Кому предназначено ПО

Программное обеспечение «Хайперплатформ» рассчитано на три категории пользователей.

Администраторы

Управление компаниями, ботами, пользователями.

Разработчики

Создание и настройка ботов; программирование логики взаимодействия; настройка команд и сценариев.

Пользователи

Взаимодействие с ботами через текстовые сообщения; отправка и получение файлов; работа с интерактивными формами и кнопками.

Функциональные характеристики

Что умеет «Хайперплатформ»

Полный перечень функциональных возможностей программного обеспечения. Подробное описание — в документе «Функциональные характеристики ПО» (раздел «Документация»).

Управление пользователями

  • Создание пользователя — добавление с указанием email, привязка к компании.
  • Просмотр списка — список пользователей компании, пагинация, фильтрация по статусу активности.
  • Деавторизация — отзыв всех активных сессий, принудительный выход из системы.
  • Удаление — полное удаление из БД с каскадным удалением чатов и сообщений.

Управление ботами

  • Создание бота — наименование и описание, привязка к компании, авто-генерация токена, настройка команд (/start, /help), загрузка аватара и фона.
  • Редактирование — изменение наименования, описания, аватара, фона, статуса активности и команд.
  • Просмотр списка — список ботов компании, пагинация и фильтрация.
  • Управление токеном — просмотр текущего, генерация нового (старый перестаёт работать).
  • Удаление — деактивация бота с сохранением истории чатов.

Управление чатами

  • Создание чата — между ботом и пользователем, автоматическая проверка существования.
  • Просмотр списка — пагинация, фильтрация по боту и пользователю, отображение email.
  • Удаление чата — удаление чата и всех связанных сообщений.

Обмен сообщениями

  • Текстовые сообщения — отправка и получение, HTML-форматирование, авто-замена переносов строк.
  • Медиафайлы — изображения (JPEG, PNG, GIF, WebP, HEIC), видео (MP4, AVI, MOV, MKV, WebM), аудио (MP3, WAV, OGG, M4A), документы (PDF, DOC, XLS, TXT).
  • Генерация превью — для изображений и первого кадра видео, отдельное хранение.
  • Чанковая загрузка — загрузка больших файлов по частям со сборкой на сервере.
  • Редактирование сообщений — изменение текста сообщения, замена прикреплённого файла, обновление клавиатуры.
  • Удаление сообщений — мягкое удаление (помечается как удалённое), файлы остаются на диске до очистки.
  • Очистка чата — удаление всех сообщений в чате.

Интерактивные элементы

  • Inline-клавиатуры — кнопки под сообщением, callback-данные, многоуровневая структура (ряды кнопок).
  • TextButton — кастомные текстовые кнопки, настройка цвета фона и текста, анимация мигания.
  • Формы — текстовые поля, выпадающие списки, радио-кнопки, чекбоксы, дата и время; обязательность и предзаполнение полей.

Аутентификация и безопасность

  • Аутентификация пользователей — вход по PIN-коду на email, ограничение попыток и времени действия.
  • Управление сессиями — JWT-токены, хранение в БД, авто-истечение срока, отзыв при деавторизации.
  • Авторизация компаний — Bearer-токены для доступа к административным функциям, проверка прав при каждом запросе.
  • Авторизация ботов — Bearer-токены для доступа к API бота, ограничение доступа только к своим чатам.
  • Безопасность cookies — HTTP-only, Secure (HTTPS), SameSite-политика (защита от CSRF).

Real-time уведомления

  • WebSocket-соединение — постоянное соединение с сервером, аутентификация через session cookie.
  • Типы уведомлений — новое сообщение, редактирование, удаление, очистка чата.

Получение обновлений

  • Long polling с таймаутом до 30 секунд.
  • Ограничение количества возвращаемых обновлений.
  • Возможность сброса очереди.

Очередь обновлений

  • Хранение непрочитанных обновлений в кэше.
  • Автоматическая маркировка прочитанных.
  • Блокировка одновременных запросов от одного бота.

Работа с файлами

  • Хранение файлов — раздельное хранение оригиналов и превью, уникальные имена файлов на основе ID сообщения, поддержка вложенных папок по chat_id.
  • Мониторинг дискового пространства — автоматическое удаление старых удалённых сообщений (старше 24 часов), аварийная очистка при превышении лимита, приоритетное удаление сообщений с файлами.

Очистка базы данных

  • Удаление записей о сообщениях после удаления файлов.
  • Логирование результатов очистки.
